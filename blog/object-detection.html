Object Detection, YOLO & Segmentation:
Where Is It, and What Exactly Is It?

Classification says "there's a person in this image." Detection says "there's a person at coordinates (x, y, w, h)." Segmentation says "these exact pixels are a person." Three questions, three levels of spatial understanding — and YOLO made the hardest real-time problem solvable in a single forward pass.

01 — Three Tasks, Three Levels of Spatial Understanding

Computer vision has three fundamental output types. Each requires a different output format, a different loss function, and a different amount of annotation effort to train. Understanding the hierarchy makes the whole field make sense.

Image classification (Days 15–17) produces a single label for the entire image. Input: 224×224 pixels. Output: "cat" (probability 0.97). The spatial location of the cat is irrelevant — the label is global. One forward pass, one vector, one argmax.

Object detection produces a label and a bounding box for each object present. Output: [("person", x=120, y=80, w=60, h=140, conf=0.91), ("car", x=340, y=200, w=180, h=90, conf=0.87)]. The model must simultaneously classify and localise — two tasks in one network, with a loss function that penalises both wrong labels and imprecise boxes.

Segmentation assigns a class label to every single pixel. Semantic segmentation gives each pixel a class ("road," "sky," "person") without distinguishing between individuals. Instance segmentation goes further: it distinguishes the pedestrian on the left from the pedestrian on the right, giving each a separate per-pixel mask. Panoptic segmentation combines both.

🗺️ Analogy: Three Ways to Describe a City

Classification is like answering "what country is this city in?" Detection is like drawing labelled rectangles around every building on a map. Semantic segmentation is like colour-coding every square metre — residential red, commercial blue, park green — without naming individual buildings. Instance segmentation is that same colour-coded map but also numbering each individual building separately so you can refer to "Building 47."

Each level requires more annotation effort, more compute, and unlocks different downstream applications. A tumour detector needs bounding boxes; a surgical robot needs pixel masks.

02 — The Architecture Timeline: How We Got Here

Computer vision didn't leap from "no CNN" to "YOLOv8 detects 80 classes at 160 FPS." It evolved through a chain of papers, each fixing a specific bottleneck in the previous one. Understanding the lineage tells you why each architecture looks the way it does — and which problems are still open.

Three parallel tracks evolved simultaneously: the backbone CNNs that extract features (shared across all tasks), the detection architectures built on top of them, and the segmentation architectures that repurposed the same backbones for pixel-level output.

🧱 Backbone CNNs  — Feature extractors shared across all tasks
2012
AlexNet Leap
The ignition point. First deep CNN to win ImageNet by a large margin (top-5 error: 15.3% vs 26.2% for the runner-up). Introduced ReLU activations (replacing sigmoid → much faster training), dropout regularisation, and GPU-parallel training. Proved deep networks work at scale. Every architecture since is a direct descendant.
2014
VGG-16 / VGG-19 Solid
Depth over width. Replaced large filters (11×11, 5×5) with stacked 3×3 convolutions — two 3×3 layers have the same receptive field as one 5×5 but fewer parameters and more non-linearity. Showed that depth (16–19 layers) matters more than filter size. Still used as a backbone for transfer learning because its features are clean and predictable. Weakness: 138M parameters, memory-heavy.
2015
ResNet-50/101/152 Leap
Residual connections solved the vanishing gradient problem. Before ResNet, training networks deeper than ~20 layers made accuracy worse — gradients decayed to near-zero before reaching early layers. ResNet's fix: add a shortcut connection that adds the input directly to the output (F(x) + x). Gradients now have a direct highway back. ResNet-50 (25M params) achieves lower error than VGG-19 (138M params). ResNet backbones are the default starting point for detection and segmentation today.
2017
DenseNet Solid
Every layer connects to every subsequent layer. Where ResNet adds input to output (x + F(x)), DenseNet concatenates all previous feature maps: each layer receives the outputs of all preceding layers. This maximises feature reuse and enables very deep networks with fewer parameters. Particularly strong in medical imaging segmentation (dense feature reuse helps recover spatial detail). Less common in detection — the concatenation overhead is high at scale.
2019
EfficientNet-B0…B7 Leap
Compound scaling: simultaneously scale depth, width, and resolution. Previous architectures scaled one dimension at a time (make it deeper, or wider, or use bigger images). EfficientNet used Neural Architecture Search to find the optimal base model, then defined a principled scaling rule: scale all three dimensions together with a fixed ratio. EfficientNet-B4 matches ResNet-152 accuracy at 5.3× fewer parameters. The B-suffix variants are the standard backbone for transfer learning on limited data — your brain tumour classifier (Day 17's fast food model) used EfficientNet.
2022
ConvNeXt Solid
A pure CNN that matches Vision Transformers. Built by systematically applying ViT design choices (larger kernels, depthwise convolutions, fewer activation functions, LayerNorm instead of BatchNorm) to a ResNet base. ConvNeXt-L matches Swin Transformer on ImageNet without any attention mechanism. Relevant because it demonstrated that CNNs hadn't hit a ceiling — the architectural improvements came from design choices, not the attention mechanism itself.
🟩 Object Detection  — Two-stage then one-stage
2014
R-CNN Pioneering
First CNN-based detector. Pipeline: (1) Selective Search proposes ~2,000 candidate regions, (2) each region is warped to 227×227 and passed through AlexNet independently, (3) SVM classifies. Achieved 53.3% mAP on PASCAL VOC — huge improvement over pre-deep methods. Fatal flaw: 2,000 CNN forward passes per image = 47 seconds/image. Not deployable.
2015
Fast R-CNN Solid
Fixed the duplicate CNN passes. Key insight: run the backbone CNN once on the whole image to produce a feature map, then project each proposed region onto that shared feature map using RoI Pooling. Now there's 1 forward pass for the backbone, not 2,000. Classification and box regression are unified into one network (no separate SVM). Speed: 0.3 seconds/image. Still limited by Selective Search taking 2 seconds per image — a non-deep bottleneck.
2015
Faster R-CNN Leap
Replaced Selective Search with a learned Region Proposal Network (RPN). The RPN shares convolutional features with the detection head — the backbone runs once, proposals and detection both use the same feature map. End-to-end trainable for the first time. Speed: ~200ms (5 FPS). This is still the standard two-stage architecture. Accuracy ceiling: ~47 mAP on COCO. Used as Mask R-CNN's backbone in 2017 for instance segmentation.
2015
YOLO v1 Leap
Reframed detection as regression, not classification. Divide image into 7×7 grid; each cell predicts 2 boxes + class probabilities directly. One forward pass, all predictions simultaneously. Speed: 45 FPS real-time. Accuracy: 63.4% mAP on PASCAL VOC — competitive with two-stage methods at 10× the speed. Weakness: struggles with small objects and clustered objects (each cell could only predict one class).
2016
SSD Solid
Multi-scale one-stage detection. Unlike YOLO v1's single grid, SSD predicts at multiple feature map resolutions simultaneously — large feature maps for small objects, small feature maps for large objects. Uses predefined anchor boxes at each scale. Faster and more accurate on small objects than YOLO v1. SSD's multi-scale prediction idea was absorbed into all subsequent YOLO versions via the Feature Pyramid Network.
2016–18
YOLO v2 → v3 Solid
v2: Added anchor boxes (borrowed from Faster R-CNN), batch normalisation, higher resolution training. mAP: 78.6% VOC. v3: Added multi-scale prediction at 3 resolutions (like SSD), darknet-53 backbone (residual connections), binary cross-entropy per class (enabling multi-label). mAP: 57.9% COCO at 51 FPS. v3 remained the production standard for years due to its balance of speed and accuracy.
2020
YOLO v4 / v5 Solid
Engineering excellence. v4 introduced CSPNet (Cross Stage Partial Networks) backbone, PANet neck for better multi-scale feature fusion, mosaic augmentation (4 images combined), and CIoU loss. v5 (Ultralytics) added auto-anchor computation, auto-learning of augmentation parameters, and streamlined PyTorch implementation. Both pushed accuracy above 50 mAP@[.5:.95] COCO while maintaining real-time speed.
2022
YOLO v6 / v7 / YOLOX Refinements
Multiple competing implementations improved anchor-free heads, dynamic label assignment (replacing fixed anchor matching), and re-parameterisation for faster inference. YOLOX introduced fully anchor-free YOLO for the first time. These refinements fed directly into v8.
2023
YOLO v8 Current Standard
Anchor-free, unified, production-grade. Dropped all anchor boxes — predicts box centre + size directly. Unified API for detection, segmentation, pose estimation, and classification in one library. C2f backbone (CSP with 2 bottlenecks). Distribution Focal Loss for more accurate box regression. YOLOv8n: 3.2M params, 160+ FPS. YOLOv8x: 68M params, ~53 mAP COCO. The current industry default.
🟨 Segmentation  — Pixel-level understanding
2014
FCN — Fully Convolutional Network Pioneering
First end-to-end CNN trained for dense pixel prediction. Replaced the fully connected layers at the end of classification CNNs (AlexNet, VGG) with convolutional layers, allowing arbitrary input sizes and producing spatial output maps. Used transposed convolutions (learnable upsampling) to restore resolution. Weakness: upsampling is coarse — boundaries are blurry because spatial detail from the encoder is discarded.
2015
U-Net Leap
Skip connections fixed FCN's blurry boundaries. The encoder-decoder with skip connections at every resolution level — high-resolution encoder features bypass the bottleneck and concatenate with decoder features at the matching scale. Designed for biomedical images with very limited training data; introduced the data augmentation strategy (elastic deformations) that became standard for medical imaging. Still the default architecture for medical segmentation after 10 years.
2016–18
DeepLab v1–v3+ Leap
Atrous (dilated) convolutions for large receptive fields without losing resolution. Standard convolutions with stride 2 downsample the feature map. Atrous convolutions expand the receptive field by inserting gaps between kernel weights — seeing a wider context without reducing spatial resolution. DeepLab v3+ added the ASPP (Atrous Spatial Pyramid Pooling) module which applies multiple dilation rates simultaneously, capturing objects at multiple scales. Standard for outdoor scene segmentation (Cityscapes, ADE20K).
2017
Mask R-CNN Leap
Added a mask branch to Faster R-CNN for instance segmentation. For each proposed region, a lightweight FCN head predicts a 28×28 binary mask in parallel with the existing classification and box regression heads. The key fix over naive RoI Pooling: RoIAlign (bilinear interpolation instead of integer quantisation) preserves sub-pixel spatial precision needed for accurate masks. The reference implementation for instance segmentation — all subsequent instance segmentation models compare to it.
2020
nnU-Net Leap
"No-new-Net" — automatic self-configuring U-Net. The key insight: for medical segmentation, the U-Net architecture is already nearly optimal. What varies is the configuration — patch size, batch size, normalization, augmentation strength — based on dataset properties (image spacing, class distribution, size). nnU-Net automatically configures all hyperparameters from dataset statistics. Outperformed all task-specific methods on 23 out of 23 medical segmentation benchmarks without task-specific tuning. The production standard for medical image segmentation.
2023
SAM — Segment Anything Foundation Model
Segmentation as a foundation model. Trained on 1 billion masks (semi-automatically generated). Accepts a point, box, or text prompt and segments any object zero-shot. Vision Transformer encoder + prompt encoder + lightweight mask decoder. SAM2 extends to video with a memory bank that propagates masks across frames. The shift from "train a segmentation model per task" to "prompt a universal segmentation model" mirrors the shift from task-specific classifiers to GPT.

Do Detection and Segmentation Use the Same Backbone CNNs?

This is a crucial practical question. The short answer: yes, the backbone is almost always shared — but the head is task-specific. The backbone's job is feature extraction, which is the same regardless of what the features are used for. Only the output head (what you do with those features) changes per task.

🟣 Universal Backbones — All Three Tasks
ResNet-50/101 and EfficientNet-B0…B7 are used as the backbone for classification, detection, and segmentation. When you load yolov8s.pt, its backbone is a modified CSP-ResNet. When you load a U-Net for medical imaging, you typically replace its encoder with a pretrained ResNet-50. The same weights that classify ImageNet dogs also extract edge and texture features useful for detecting pedestrians and segmenting tumours.
🟩 Detection-Specific Heads
After the shared backbone, detection adds a neck (Feature Pyramid Network — multi-scale feature fusion) and a head (predicts boxes + classes). The RPN in Faster R-CNN and the grid-based prediction head in YOLO are detection-specific. The backbone (ResNet, DarkNet, C2f) is interchangeable. Swapping backbone: YOLO('yolov8s.pt') → replace backbone with ConvNeXt → same head, different features.
🟨 Segmentation-Specific Heads
After the shared backbone encoder, segmentation adds a decoder (gradually upsamples back to full resolution). U-Net's decoder with skip connections, DeepLab's ASPP module, Mask R-CNN's mask branch — all are segmentation-specific. The encoder (ResNet-50 pretrained on ImageNet) is identical. A ResNet-50 backbone can simultaneously power a YOLO detector and a U-Net segmentor — you'd just connect different heads.
🔵 Where Backbones Diverge
Some backbones are optimised for specific tasks: DarkNet (YOLO's custom backbone) is optimised for detection speed. U-Net's original encoder was a simple VGG-style encoder without pretrained weights (2015 medical data was scarce). For production medical segmentation today, you always replace it with a pretrained ResNet-50 or EfficientNet — the shared backbone principle wins over task-specific design.

03 — How Object Detection Works: From R-CNN to YOLO

The fundamental challenge in detection: you don't know in advance how many objects are in an image, where they are, or how large they are. You need a network that outputs a variable-length list of (box, class, confidence) tuples from a fixed-size input.

The classic two-stage approach — the R-CNN family — solved this in two steps. First, a region proposal network identifies ~2,000 candidate patches that might contain objects. Second, a CNN classifies each patch. R-CNN (2014) was accurate but processed one image in 47 seconds. Fast R-CNN got this to seconds by sharing convolutional features. Faster R-CNN got to ~200ms by making the region proposal network part of the same network. But "two stages, two passes" had a ceiling on speed that no engineering could overcome.

YOLO (You Only Look Once, 2015) took a fundamentally different approach: frame detection as a single regression problem. Divide the image into a grid. Have each grid cell directly predict boxes and class labels simultaneously. One network, one pass, all detections at once. The original YOLO ran at 45 FPS — real-time detection for the first time. YOLOv8 (2023) exceeds 160 FPS on modern hardware.

⚡ The Speed–Accuracy Tradeoff

Two-stage detectors (Faster R-CNN, Mask R-CNN) remain more accurate on small objects and dense scenes because their region proposals are more precise. YOLO-family one-stage detectors are faster because they skip the proposal step entirely. For real-time applications (autonomous driving, live video), YOLO wins. For offline analysis of medical images where every pixel matters, two-stage or segmentation models win. Modern benchmarks on COCO: YOLOv8x reaches ~53 mAP; DINO (transformer-based two-stage) reaches ~63 mAP, at ~10× the inference cost.

04 — Inside YOLO: How One Forward Pass Detects Everything

YOLO's architecture has three main components: a backbone (feature extraction), a neck (multi-scale feature fusion), and a head (final predictions). Here's how each works:

1
Grid division. The input image (e.g., 640×640) is divided into an S×S grid. In YOLOv8, three different grid sizes are used simultaneously: 80×80 (for small objects), 40×40 (medium), 20×20 (large). Each grid cell is responsible for detecting objects whose centre falls within that cell. This is why YOLO misses nothing — every spatial location is covered by some cell.
2
Anchor-free prediction (YOLOv8). Earlier YOLO versions used predefined anchor boxes — fixed aspect-ratio templates (tall-thin for people, wide-short for cars) from which the network predicted offsets. YOLOv8 dropped anchors entirely. Each cell directly predicts the box centre (cx, cy), width and height (w, h) as fractions of the grid cell, plus class probabilities. Simpler, more general, better on unusual aspect ratios.
3
Multi-scale detection (FPN neck). A Feature Pyramid Network fuses features from multiple backbone layers at different resolutions. The 80×80 grid sees fine spatial detail — good for small objects. The 20×20 grid sees large receptive fields and semantic content — good for large objects. All three scales are processed simultaneously and their predictions merged, which is what lets YOLO handle cars 100m away (tiny in image) and trucks 5m away (giant in image) in the same pass.
4
Confidence thresholding. Each prediction has an objectness score and per-class probabilities. The final confidence = objectness × class_probability. Predictions below a confidence threshold (typically 0.25–0.4) are discarded immediately. This is fast — a simple multiplication and comparison, applied to thousands of candidate boxes in parallel on GPU.
5
Non-Maximum Suppression (NMS). After thresholding, many overlapping boxes remain for each object — the network found the person at (x=120, conf=0.92) and also at (x=122, conf=0.87) and (x=119, conf=0.81). NMS keeps only the highest-confidence box and suppresses all others with IoU above a threshold (typically 0.45). Without NMS, every detected object would be reported 10–20 times.
📐 IoU: The Core Detection Metric

Intersection over Union (IoU) = area of overlap between predicted box and ground-truth box ÷ area of their union. A perfect prediction has IoU = 1.0. Complete miss has IoU = 0. IoU > 0.5 is the conventional threshold for a "correct" detection in standard benchmarks. IoU appears in three places in YOLO: the loss function (penalise imprecise boxes), confidence thresholding (is there actually an object here?), and NMS (are these two boxes covering the same object?).

The COCO benchmark evaluates mAP at IoU thresholds from 0.5 to 0.95 in 0.05 steps and averages them — written as AP@[.5:.95]. This is much harder than the older PASCAL VOC metric, which is why COCO scores look lower.

05 — The Detection Loss Function

Object detection training minimises a composite loss with three independent components, each penalising a different type of error:

Localisation loss
Penalises inaccurate bounding box predictions. YOLOv8 uses CIoU loss — a variant that simultaneously penalises overlap area, centre distance, and aspect ratio difference. Why not just MSE? Two boxes with the same MSE error can have wildly different IoU. A box that's slightly off-centre but correctly sized is far better than one that's correctly centred but wrong size — CIoU captures this properly.
Objectness loss
Penalises errors in predicting whether a grid cell contains an object at all. Binary cross-entropy: 1 if an object's centre falls in this cell, 0 otherwise. In a typical scene, >95% of grid cells contain no object — this class imbalance is why objectness loss is weighted lower than classification loss during training.
Classification loss
Penalises wrong class predictions for cells that do contain an object. Binary cross-entropy per class (not softmax), which allows multi-label predictions — a bounding box can simultaneously be "person" and "athlete" without forcing the model to choose one. YOLOv8 uses distribution focal loss to focus training on hard examples.

The three losses are summed with weights: total_loss = λ_box × L_box + λ_obj × L_obj + λ_cls × L_cls. In YOLOv8, default weights are approximately box=7.5, obj=0.5, cls=0.5. These ratios reflect that box regression is the hardest sub-task and deserves the most training signal. Getting these weights wrong (e.g., obj loss too high) causes the model to predict "something is here" confidently but fail to classify or localise it correctly.

06 — Code: Running and Fine-Tuning YOLOv8

The Ultralytics library makes YOLOv8 inference and fine-tuning remarkably concise. Below is a complete pipeline from off-the-shelf inference to fine-tuning on a custom dataset, with all the details that matter in practice.

# pip install ultralytics
from ultralytics import YOLO
import cv2, torch

# ── Model variants ────────────────────────────────────────────────────────
# yolov8n: nano  — 3.2M params,  ~160 FPS  (embedded/mobile)
# yolov8s: small — 11M params,   ~120 FPS
# yolov8m: med   — 25M params,   ~80 FPS
# yolov8l: large — 43M params,   ~50 FPS
# yolov8x: xlarge— 68M params,   ~30 FPS, best accuracy
model = YOLO('yolov8n.pt')   # downloads pretrained weights automatically

# ── Inference on a single image ──────────────────────────────────────────
results = model('street.jpg', conf=0.4, iou=0.45)

for r in results:
    for box in r.boxes:
        cls_name = model.names[int(box.cls)]
        conf     = box.conf.item()
        x1, y1, x2, y2 = box.xyxy[0].tolist()
        print(f"{cls_name:12s} {conf:.2f}  [{x1:.0f},{y1:.0f} → {x2:.0f},{y2:.0f}]")

    # Draw boxes and save
    annotated = r.plot()
    cv2.imwrite('output.jpg', annotated)

# ── Fine-tune on a custom dataset ─────────────────────────────────────────
# dataset.yaml must define:
#   path: /data/my_dataset
#   train: images/train    # folder with images + labels/*.txt (YOLO format)
#   val:   images/val
#   nc:    3               # number of classes
#   names: ['car', 'person', 'cyclist']

model = YOLO('yolov8s.pt')   # start from pretrained (transfer learning)
results = model.train(
    data    = 'dataset.yaml',
    epochs  = 50,
    imgsz   = 640,
    batch   = 16,
    lr0     = 0.01,       # initial LR — decays via cosine schedule
    patience= 10,       # early stopping: stop if no improvement for 10 epochs
    device  = 0,          # GPU 0; use 'cpu' if no GPU
    augment = True,       # mosaic, mixup, hsv jitter, flips — all on by default
    name    = 'my_detector'
)

# ── Evaluate on test set ─────────────────────────────────────────────────
metrics = model.val(data='dataset.yaml', split='test')
print(f"mAP50:    {metrics.box.map50:.3f}")
print(f"mAP50-95: {metrics.box.map:.3f}")

07 — Segmentation: Pixel-Level Understanding

Segmentation networks must output a prediction for every pixel — so the output must match the input's spatial dimensions. The standard architecture is an encoder-decoder: the encoder (a CNN or transformer backbone) compresses the image into a compact high-level representation, and the decoder gradually upsamples it back to full resolution, predicting a class label at each pixel along the way.

The problem with naive upsampling: the encoder's compression destroys spatial detail. Knowing "there is a tumour somewhere" doesn't tell you exactly where each boundary pixel falls. This is the problem U-Net (Ronneberger et al., 2015) solved.

🔗 U-Net: The Architecture That Defined Medical Segmentation

U-Net introduced skip connections between matching encoder and decoder layers. At each resolution level, the decoder receives two inputs: the upsampled features from the level below (semantic understanding — what the object is) and the high-resolution features from the corresponding encoder layer (spatial precision — exactly where each pixel boundary falls). By concatenating these, the decoder can produce sharp boundaries that neither pure downsampling nor pure upsampling could achieve alone.

The "U" in U-Net refers to the architecture's shape: encoder contracts left-to-right (down the left side of the U), skip connections cross horizontally, decoder expands right-to-left (down the right side). U-Net and its descendants (nnU-Net, Swin-UNet, MedSAM) remain the backbone of clinical medical image analysis worldwide.

Semantic segmentation
Every pixel gets a class label. Two pedestrians walking side by side are both labelled "person" — no distinction between individuals. U-Net, DeepLab, SegNet. Use case: road scene understanding, satellite land-cover mapping.
Instance segmentation
Each object instance gets its own unique pixel mask. The pedestrian on the left is "person #1," the one on the right is "person #2." Mask R-CNN (2017) added a mask prediction branch to Faster R-CNN. Use case: cell counting in microscopy, surgical tool tracking.
Panoptic segmentation
Combines semantic + instance: "stuff" classes (sky, road) get global semantic labels; "thing" classes (people, cars) get individual instance masks. Provides a complete scene description. Used in autonomous driving scene understanding pipelines.
YOLOv8-seg
YOLOv8 supports instance segmentation natively via a mask prediction head alongside the detection head. Uses the same single-pass architecture: detection boxes + lightweight prototype masks, merged at inference. YOLO('yolov8n-seg.pt') — add -seg suffix.

08 — Metrics: How You Know If It's Working

Detection and segmentation use different metrics because their outputs differ. Detection boxes are continuous (you can be "close" to the right box); segmentation masks are binary per-pixel (you either got the pixel right or you didn't). The metrics reflect this.

MetricFormulaUsed forNotes
IoU / Jaccard |A ∩ B| / |A ∪ B| Detection & segmentation 0 = no overlap, 1 = perfect. Threshold 0.5 standard for detection "hit"
Dice / F1 2|A ∩ B| / (|A| + |B|) Segmentation (esp. medical) More sensitive to small regions than IoU. Preferred for small lesions, thin structures
AP@50 Area under PR curve at IoU=0.5 Detection (PASCAL VOC style) Older, easier benchmark. 80%+ on VOC is competitive
mAP@[.5:.95] Mean AP over IoU 0.5→0.95 Detection (COCO style) Harder, more comprehensive. 50+ is competitive on COCO 80-class
FPS Frames per second Real-time applications >30 FPS for real-time. 5–10 FPS for offline/batch analysis is acceptable
⚠️ Why Dice Beats IoU for Medical Segmentation

Consider a 512×512 MRI slice where a small lesion occupies 200 pixels out of 262,144 total. A model that predicts "no lesion anywhere" gets: IoU = 0/200 = 0 (correct — no overlap), but pixel accuracy = 262,144−200/262,144 = 99.9% (misleadingly perfect). Dice explicitly penalises missing small regions: if you miss 200 lesion pixels, Dice = 0. For medical imaging where small structures are clinically critical, Dice is the right metric. Never report pixel accuracy alone on imbalanced segmentation tasks.

09 — Why These Architectures Still Matter

These aren't dated techniques from a previous era of deep learning. They're canonical patterns that appear inside the systems you interact with daily.

Autonomous vehicles cannot function without real-time detection. A car doing 100 km/h needs to detect and localise pedestrians, cyclists, and vehicles in under 33 milliseconds per frame. YOLO's single-pass architecture makes this possible on embedded hardware. Tesla's Autopilot, Waymo's driver, and every production AV system runs detection networks in this family.

Medical imaging depends on segmentation. Tumour volume measurement, organ boundary delineation, lesion tracking over time — all require pixel-level masks, not bounding boxes. U-Net and its descendants (nnU-Net, Swin-UNet, MedSAM) are the backbone of clinical analysis pipelines in radiology and pathology worldwide. The diabetic retinopathy project from Day 17 uses segmentation as part of its grading pipeline.

The U-Net architecture lives inside generative AI. Stable Diffusion's denoising network is literally a U-Net. The encoder-decoder structure with skip connections that Ronneberger designed for medical image segmentation in 2015 is the same architecture diffusing latent representations in every image-generation model. When you generate an image with Midjourney or DALL-E, the diffusion step runs a U-Net.

Instance segmentation powers generative AI evaluation. When diffusion models generate images, quality evaluation uses segmentation models to check whether generated objects have physically plausible boundaries and spatial relationships. Detection and segmentation are embedded in the evaluation pipelines of the models you use daily.

🔗 SAM: When Segmentation Became a Foundation Model

The Segment Anything Model (SAM, Meta 2023) generalised segmentation into a foundation model: trained on 1 billion masks, it can segment any object in any image given a point, a box, or a text prompt — zero-shot, no fine-tuning. SAM2 extended this to video with a memory mechanism that propagates masks across frames. This is the vision equivalent of GPT: a single large model replacing many specialised segmentation models. SAM's architecture (Vision Transformer encoder + prompt encoder + mask decoder) uses the same attention mechanism you'll meet in Day 20 — vision and language have largely converged on the same fundamental building block.

🧠 Check Your Understanding
1. What is the fundamental architectural difference between two-stage detectors (Faster R-CNN) and one-stage detectors (YOLO) — and what does each trade off?
Two-stage detectors first run a Region Proposal Network to generate ~2,000 candidate object regions, then classify and refine each region separately — two forward passes, two stages. This gives them high accuracy because the proposal stage focuses the classifier on likely object locations, producing precise boxes even for small or densely-packed objects. One-stage detectors like YOLO skip region proposals entirely: a single network directly predicts all bounding boxes and classes for all grid cells in one forward pass. This is 10–30× faster but historically less accurate on small or overlapping objects. The tradeoff: two-stage for maximum accuracy in offline analysis, one-stage for real-time applications where speed is non-negotiable.
2. Explain Non-Maximum Suppression step by step. What problem does it solve, and what goes wrong if you set the IoU threshold too high or too low?
The problem: YOLO predicts a box for every grid cell, producing many overlapping detections for each object. NMS step by step: (1) sort all predictions by confidence score, descending; (2) take the highest-confidence box — this is a keeper; (3) compute IoU between this keeper and every remaining box of the same class; (4) suppress (remove) all boxes with IoU above the threshold — they're covering the same object; (5) repeat with the next highest-confidence remaining box. Too low threshold (e.g., 0.3): aggressively removes boxes — you suppress legitimate detections of nearby objects of the same class. A crowd of people gets collapsed to one box. Too high threshold (e.g., 0.8): barely suppresses anything — the same person gets detected 15 times with slightly different boxes. Typical good value: 0.4–0.5.
3. Why does U-Net use skip connections, and why can't the encoder-decoder without them achieve the same segmentation quality?
Encoder downsampling progressively destroys spatial detail in favour of semantic abstraction. By the bottleneck, a 512×512 image might be represented as a 16×16 feature map — semantically rich but spatially vague. The decoder must reconstruct full-resolution output, but upsampling from 16×16 to 512×512 produces blurry, imprecise boundaries because the spatial information was discarded. Skip connections solve this by carrying high-resolution feature maps directly from the encoder to the corresponding decoder level — bypassing the bottleneck compression. At each decoder resolution level, the decoder concatenates the upsampled coarse features (what the object is) with the skip-connected fine features (exactly where each pixel boundary falls). Without skip connections, the decoder has no access to the precise spatial locations that were encoded before downsampling, so boundary predictions are blurry. With them, the decoder reconstructs sharp, accurate segmentation masks.
4. Why is Dice score preferred over pixel accuracy for medical image segmentation tasks involving small structures?
Pixel accuracy is misleading when classes are heavily imbalanced — which is almost always the case in medical imaging. A 512×512 MRI slice where a lesion occupies 200 pixels out of 262,144 total: a model predicting "background everywhere" achieves 99.9% pixel accuracy while completely missing the lesion. Dice = 2|A∩B| / (|A|+|B|) explicitly measures overlap between predicted and ground-truth masks. If you predict nothing, Dice = 0/200 = 0, correctly flagging complete failure. Dice is also symmetric (it treats false positives and false negatives equally), more sensitive to small regions than IoU (since it uses |A|+|B| in the denominator rather than |A∪B|, which is dominated by the large background class). For any task where the clinically important object is small relative to the image — lesions, polyps, microaneurysms — Dice is the right metric.
5. The U-Net architecture appears inside Stable Diffusion. What is the connection — and why does the same encoder-decoder-with-skip-connections design work for both segmentation and image generation?
Stable Diffusion's core operation is iterative denoising: given a noisy latent image at timestep t, predict the noise that was added so it can be subtracted. This requires the network to understand both high-level semantic content (what object is at what location — needs a large receptive field, encoded by the encoder) and fine spatial detail (exactly what texture and boundary to render — needs high-resolution features). This is the same dual requirement that U-Net was designed for in segmentation: semantic understanding from the bottleneck + spatial precision from skip connections. The encoder-decoder-with-skip structure is the canonical solution to "I need both global semantic context and local spatial precision simultaneously" — which turns out to be a requirement of both segmentation and diffusion denoising. The architecture generalises because the underlying problem structure is the same, even though the tasks superficially look completely different.
⚡ Quick Recap — Object Detection & Segmentation
  1. Three levels of spatial understanding: classification (global label) → detection (label + bounding box) → segmentation (label per pixel). Each requires more annotation effort and produces richer spatial output.
  2. Two-stage detectors (Faster R-CNN) propose regions first, then classify — accurate but slow. One-stage detectors (YOLO) predict everything in one pass — fast enough for real-time.
  3. YOLO's key insight: frame detection as regression. Divide image into a grid; each cell directly predicts boxes and class labels. No region proposals. One forward pass, all detections simultaneously.
  4. Multi-scale detection (FPN neck) lets YOLO detect small objects in fine grids (80×80) and large objects in coarse grids (20×20) in the same pass. This is what handles far-away and close-up objects together.
  5. Non-Maximum Suppression (NMS) removes duplicate detections: keep the highest-confidence box, suppress all overlapping boxes (IoU above threshold) for the same class. Without NMS, every object would be reported 10–20 times.
  6. IoU = intersection area / union area. Used in loss, confidence thresholding, NMS, and evaluation means a detection is "correct" if IoU with ground truth > 0.5).
  7. U-Net's innovation = skip connections between encoder and decoder layers. Semantic understanding from the bottleneck + spatial precision from skip connections = sharp segmentation boundaries. Original 2015, still state of art in medical imaging.
  8. Dice score is the right metric for medical segmentation — it's not fooled by background dominance. Pixel accuracy can reach 99.9% by predicting only background. Dice = 0 in the same case.
  9. U-Net is literally inside Stable Diffusion. The denoising network is a U-Net. The same encoder-decoder-with-skip design works for both segmentation and image generation because both need global semantics + local spatial detail simultaneously.
  10. SAM (2023) generalised segmentation into a foundation model: segment any object from a point, box, or text prompt — zero-shot. The vision equivalent of GPT, bridging the gap between computer vision and language models.
🎯 Interview Questions
  1. What is the difference between semantic and instance segmentation?
    Semantic segmentation labels every pixel with a class but treats all instances identically — two people are both "person." Instance segmentation assigns a unique mask to each individual object — person #1 on the left, person #2 on the right. Instance segmentation is harder (requires knowing how many objects there are and their individual boundaries) and produces more useful output for tasks like cell counting, surgical tool tracking, or crowd analysis.
  2. What is IoU and why does it appear in so many places in detection pipelines?
    IoU (Intersection over Union) = area of overlap / area of union between two regions. It appears in: (1) the training loss — CIoU penalises imprecise predicted boxes; (2) confidence thresholding — does a predicted box sufficiently overlap a real object?; (3) Non-Maximum Suppression — are two boxes covering the same object (high IoU) or different ones?; (4) evaluation — a detection is "correct" if IoU with ground truth exceeds 0.5. It's the universal measure of spatial overlap quality, which is exactly what detection accuracy depends on.
  3. Why is YOLO called "You Only Look Once," and what does that mean architecturally?
    Unlike two-stage detectors that process the image twice (once for region proposals, once for classification), YOLO processes the image in a single forward pass and outputs all bounding boxes and class labels simultaneously. "One look" = one inference = all detections at once. This single-pass design is why YOLO achieves real-time speeds — 45 FPS for the original YOLO in 2015, 160+ FPS for YOLOv8n today — that two-stage detectors cannot match.
  4. Why does U-Net use skip connections between encoder and decoder?
    Encoder downsampling progressively loses spatial detail. The decoder reconstructing full-resolution predictions needs both the semantic understanding from the deep bottleneck (what the object is) and the precise spatial information from earlier encoder layers (where each boundary pixel falls). Skip connections carry high-resolution encoder feature maps directly to matching decoder levels, enabling sharp segmentation boundaries that upsampling alone cannot produce.
  5. When would you use Dice score instead of accuracy for segmentation evaluation?
    Whenever the class distribution is heavily imbalanced — which is almost always the case in medical imaging. A model predicting "background" for every pixel achieves 99.9% accuracy when the lesion covers <0.1% of the image. Dice = 2|A∩B|/(|A|+|B|) is 0 in this case, correctly identifying complete failure. Use Dice (or IoU) whenever the clinically or semantically important region is small relative to the total image area.
