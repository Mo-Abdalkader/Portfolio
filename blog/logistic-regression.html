Logistic Regression:
Teaching a Line to Think in Probabilities

One small mathematical trick — the sigmoid function — transforms a line into a probability machine. This 70-year-old algorithm still runs in production at every major tech company.

← Back to Blog ☰ Categories

A doctor looks at a patient's blood test results.
Cholesterol: 240. Blood pressure: 150/95. Age: 58. Smoker: yes.
She doesn't say: "Your heart disease score is 3.7."
She says: "There's an 80% chance you'll have a cardiac event in the next 10 years."

That's not linear regression. That's logistic regression.

01 — The Problem With Using Linear Regression for Classification

Suppose you want to predict: will this email be spam (1) or not spam (0)? You could try linear regression. The problem: linear regression outputs any number — it might predict 2.3 for a very spammy email, or -0.8 for a clean one. These are not probabilities. They don't even make sense as outputs for a yes/no question.

We need a model that outputs a number between 0 and 1 — interpretable as a probability. Logistic regression does exactly this by wrapping the linear model inside one clever function.

🌡️ Analogy

Think of a dimmer switch. Linear regression is a regular switch that can go from -100 to +100. You need something that, no matter how extreme the input, always outputs between 0 and 1. The sigmoid function is that dimmer — it squishes any input into the range (0, 1), smoothly, without discontinuity.

02 — The Sigmoid Function

The sigmoid takes any real number and maps it to a value between 0 and 1: σ(z) = 1 / (1 + e^(-z)). When z is very negative (say, -10), σ(z) ≈ 0. When z is 0, σ(z) = 0.5. When z is very positive (say, +10), σ(z) ≈ 1.

Logistic regression computes: z = w₁x₁ + w₂x₂ + ... + b (the linear part), then applies sigmoid: probability = σ(z). The output is now always between 0 and 1, interpretable as the probability of the positive class.

03 — The Decision Boundary

When the probability crosses 0.5, the model predicts class 1. When it's below 0.5, class 0. The decision boundary is the set of input values where the probability is exactly 0.5 — the dividing line between predictions. For logistic regression, this boundary is always a straight line (or hyperplane in higher dimensions). If your classes aren't linearly separable in the input space, logistic regression will fail to perfectly separate them.

04 — How It Trains: Log-Loss (Cross-Entropy)

Logistic regression is trained with cross-entropy loss (also called log-loss). This penalizes confident wrong answers severely. If the true label is 1 and the model outputs 0.99 → tiny loss. If the model outputs 0.01 for label 1 → enormous loss. This is mathematically much better than using MSE for classification because it creates stronger gradients when the model is confidently wrong.

1
Compute linear combination z = w·x + b for all training examples.
2
Apply sigmoid to get probabilities between 0 and 1.
3
Compute cross-entropy loss between probabilities and true labels.
4
Gradient descent updates weights to reduce loss. Repeat thousands of times.

05 — From Scratch + sklearn

import numpy as np

# Sigmoid function
def sigmoid(z): return 1 / (1 + np.exp(-z))

# Toy binary dataset (tumor size → malignant?)
X = np.array([1.5,2.0,2.5,3.0,3.5,4.0,4.5,5.0])
y = np.array([0,0,0,0,1,1,1,1])

w, b, lr = 0.0, 0.0, 0.1
for _ in range(3000):
    p    = sigmoid(w * X + b)           # predicted probabilities
    w   -= lr * np.mean((p - y) * X)   # gradient of log-loss
    b   -= lr * np.mean(p - y)

# Predict: size=3.8 → probability of malignant
prob = sigmoid(w * 3.8 + b)
print(f"P(malignant | size=3.8) = {prob:.1%}")  # → ~87%

# sklearn equivalent
from sklearn.linear_model import LogisticRegression
import warnings; warnings.filterwarnings('ignore')
clf = LogisticRegression().fit(X.reshape(-1,1), y)
print(f"sklearn P(malignant | 3.8) = {clf.predict_proba([[3.8]])[0,1]:.1%}")

06 — Regularization & When to Use It

Logistic regression supports L1 (Lasso) and L2 (Ridge) regularization via sklearn's C parameter (inverse of regularization strength: smaller C = stronger regularization). L1 is great when you suspect many features are irrelevant — it zeros them out. L2 keeps all features but shrinks large weights. In spam detection with thousands of word features, L1 regularization automatically selects the few hundred words that actually matter.

🔗 Connection to Modern AI

The final layer of BERT, GPT, and most classification neural networks is a logistic regression head — a linear layer followed by sigmoid (binary) or softmax (multi-class). When you fine-tune a pre-trained model for sentiment analysis, you're essentially training a logistic regression on top of powerful learned representations. The "old" algorithm is the last piece of the most modern architectures.

07 — A Thought to Sit With

💬 For You to Consider

Logistic regression outputs a probability — say, 73% chance of fraud. But the final decision (flag or don't flag) requires a threshold, typically 50%. Moving this threshold changes the tradeoff between catching real fraud and falsely flagging legitimate transactions.

Who should decide where that threshold sits — the data scientist, the business, or the regulator? And whose interests might conflict?

🎯 Interview Questions
  1. Why can't we use linear regression for classification?
    Linear regression outputs unbounded values (can be negative or greater than 1), which don't represent probabilities. It also assumes a linear relationship in the output, which doesn't suit binary outcomes.
  2. What is the sigmoid function and why do we use it?
    Sigmoid maps any real number to (0, 1), making it suitable for outputting probabilities. It's differentiable everywhere, enabling gradient descent, and has a natural probabilistic interpretation.
  3. What loss function does logistic regression use and why?
    Cross-entropy (log-loss). It penalizes confident wrong predictions very heavily, creating strong gradients that drive faster learning than MSE would in a classification setting.
  4. What is the decision boundary in logistic regression?
    The set of input values where the model outputs a probability of exactly 0.5. For standard logistic regression, this is always a linear boundary (line in 2D, hyperplane in higher dimensions).
  5. How does the C parameter in sklearn's LogisticRegression work?
    C is the inverse of regularization strength. Small C means strong regularization (simpler model, less overfitting). Large C means weak regularization (model fits training data more closely, risk of overfitting).
